# main 工具规则

## 作用

这个文件记录 `main` 主工作区的工具使用原则。

它约束的是平台总控层面的工具使用，不是某个单独项目团队的旧根目录规则。

## 当前工具口径

- OpenClaw：多 agent 运行、管理与编排入口
- 项目工作区：项目上下文与文档沉淀入口
- `shared/`：跨项目共享规范与模板存放区
- 专业角色工作区：各 specialist agent 的独立执行区

## 使用原则

1. 工具用于推进结构化协作，不替代正式项目文档。
2. 平台级结论优先回写到根目录规则或 `shared/`。
3. 项目级结论优先回写到 `projects/<项目名>/`。
4. 角色级规则优先维护在对应 agent workspace 根目录。
5. 不把运行态状态、会话缓存、凭据或路由临时结果写进说明文件。
6. `main` 接收到外部消息时，应先判断它属于平台级事项还是某个项目事项。
7. 如果事项明显属于某个项目，可转交该项目总控角色继续处理。

## 模型与角色使用原则

- 当前角色到模型的约束主要由角色文件和项目流程文档承载。
- 真正的消息路由、agent 绑定和运行策略应最终落在 OpenClaw 配置层。
- 在未完成 routing bindings 之前，不把文档中的协作假设误认为已经自动生效。
- 在入口存在多个角色时，文档应明确“谁负责什么类型的外部消息”，不要靠临时记忆维持。
