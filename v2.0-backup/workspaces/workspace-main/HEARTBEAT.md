# 全局心跳规则

## 作用

这个文件定义 `main` 主工作区的心跳检查规则。

`main` 的心跳职责是平台级巡检、升级接收与全局整理，不负责项目细活执行。

## 每次心跳要检查的内容

1. 是否有新的待整理事项进入 `inbox/`
2. 是否有需要归档的结果仍停留在 `outbox/`
3. 是否有项目指挥官提交的升级事项尚未收口
4. 是否有跨项目或跨角色的阻塞还没有进入正式文档
5. 是否有平台级规则变更需求尚未整理
6. 是否有需要提醒你的高优先级全局事项仍未收口
7. 是否存在平台异常、规则漂移或 skill 同步问题
8. Teemo 是否上报了阻塞、长时间停工或需要你裁决的升级事项
9. 是否有应该提醒你确认的全局待决事项
10. 是否有模型调用超时/失败已超过自恢复边界，才需要升级到平台层

## 发现问题时的处理原则

### 有待整理输入

- 整理为结构化事项
- 判断属于哪个项目或哪个角色
- 必要时生成待分派说明

### 有待归档输出

- 提醒将结果写回正式项目目录或共享目录
- 不允许关键结论只停留在临时输出区

### 有项目升级事项

- 判断是否属于平台层处理
- 如果属于项目执行层，则转交给对应项目指挥官
- 如果属于平台规则层，则整理成正式变更项

### 有平台级异常或规则漂移

- 先记录异常现象、影响范围和当前状态
- 判断属于配置问题、规则问题还是跨项目冲突
- 必要时只做提醒与收口，不直接替代项目指挥执行

### 有阻塞未收口

- 汇总阻塞点
- 标明影响范围
- 如需你介入，给出明确的待确认事项

## 不做的事

- 不直接推进办公增强项目任务
- 不直接替代 Teemo 追项目细节
- 不把项目问题写回 `main` 根目录角色文件
- 不直接接管 Teemo 对 specialist 的日常催办

## 正常情况

如果没有需要你介入的事项，只返回：

`HEARTBEAT_OK`

不做无意义打扰。

## 巡检节奏建议

这里只收敛建议，不代表 gateway cron 已启用：

- 开工巡检：检查平台异常、规则漂移、全局待确认事项、Teemo 升级入口
- 收口摘要：汇总当日平台侧变化、待确认事项和需要用户知道的升级摘要
- 周期性结构检查：检查项目边界、共享目录、规则文件和长期结构是否漂移
