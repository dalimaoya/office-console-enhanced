# 用户长期偏好

这个文件用于记录你在整个 OpenClaw 多 agent 系统上的长期偏好。

这里只记录稳定偏好，不记录某个单独项目的临时要求。

## 当前已知偏好

- 你倾向于把角色独立成主 agent，而不是混在一个大工作区里。
- 你希望严格按 OpenClaw 官方规范来组织目录和工作区。
- 你希望目录、文件、角色、项目边界清晰，不要出现历史魔改遗留的耦合。
- 你偏好中文描述，尤其是工作区说明、规则文件和结构文档。
- 你重视可维护性，宁可先把结构设计清楚，再继续往上加项目和角色。
- 你接受 `main` 直接对外接收消息，也接受某些项目总控角色直接对外接收本项目消息。

## 工作方式偏好

- 先梳理结构，再做内容扩展。
- 先隔离角色，再配置路由。
- 先保住长期规则，再处理单个项目细节。
- 不希望旧状态、旧耦合、旧运行痕迹继续污染新系统。
- 不希望我擅自把外部入口强行收束成单入口模型。

## 后续记录原则

- 只补充长期稳定偏好。
- 项目专属要求不要写到这里，应写入对应项目目录。
- 角色专属要求不要写到这里，应写入对应角色工作区。
