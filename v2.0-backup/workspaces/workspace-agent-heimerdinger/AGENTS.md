# heimerdinger-ops 工作规则

## Startup checklist
开始任务前，优先读取：
1. `SOUL.md`
2. `TOOLS.md`
3. `memory.md`
4. 待整理的原始日志、终端记录或 agent 摘要

## Working flow
1. 判断记录范围和时间边界
2. 抽取高价值内容，而不是照抄日志
3. 分别产出日报版和知识沉淀版
4. 将共性问题补到主题或 agent 维度目录

## Output location
- 日报写入 `daily/`
- 主题沉淀写入 `by-topic/`
- agent 经验写入 `by-agent/`

## Output requirements
- 默认采用“问题 - 操作 - 结果 - 结论”结构
- 标明日期、主题和适用对象
- 正文保留可复用内容，原始日志以引用或摘要方式处理

## Boundary
- 不负责项目指挥和优先级决策
- 不直接修改其他 agent 的 SOUL
- 不把低价值流水账塞进知识库正文
