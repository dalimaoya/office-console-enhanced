# 控制台 v2 设计方案 — 产品视角评审

**评审人**：艾克 Ekko（产品经理）  
**日期**：2026-03-20  
**设计师**：拉克丝 Lux  
**评审对象**：方案 A（暖色结构化） vs 方案 B（指挥中心）

---

## 1. 用户场景匹配度

### 核心判断：当前控制台是"日常办公信息浏览"场景，不是"数据监控中心"

依据产品文档（product-scope-v1.md）：

- **一期用户**是李琪（项目发起人），核心验收场景是"打开控制台 10 秒内看到角色状态、阻塞项、token 消耗、最近操作"——这是**信息浏览型**使用模式，不是 7×24 盯屏的监控场景。
- 控制台定位是"项目运行的可观测性"，但一期只有 1 个项目、9 个角色，数据量级很小。
- 成功标准要求的是"一眼看清全局"和"快速排查问题"，而非实时流式刷新。

**结论**：方案 A 的日常办公导向更匹配一期场景。方案 B 的深色指挥中心视觉在当前数据密度下显得"大炮打蚊子"。

### 但方案 B 有一个强项值得保留

方案 B 的 sparkline 迷你图、`/` 快捷键搜索、"系统在线"心跳指示灯——这些交互细节在信息密度变大后（二期多项目）会非常实用。建议在方案 A 基础上吸收这三个设计元素。

---

## 2. 信息优先级分析

### 产品要求的信息使用路径

根据成功标准（场景二），用户打开控制台后的关注优先级为：

1. **角色在线状态 + 阻塞项** → 有没有问题？（P0）
2. **Token 消耗** → 成本可控吗？（P0）
3. **最近关键操作** → 刚发生了什么？（P0）
4. **项目阶段进度** → 整体走到哪了？（P1）
5. **活跃对象详情** → 具体在做什么？（P1）
6. **快速上手引导** → 一次性消费，完成后应弱化（P2）

### 两套方案的布局对比

| 信息 | 方案 A 位置 | 方案 B 位置 | 产品优先级 |
|------|------------|------------|-----------|
| KPI（含阻塞项） | 顶部 4 列 ✅ | 顶部 4 列 ✅ | P0 |
| 阻塞项详情 | 右栏顶部 ✅ | 右面板顶部 ✅ | P0 |
| 事件日志 | 右栏底部 ✅ | 右面板底部 ✅ | P0 |
| 项目进度 | 右栏中部 ✅ | 右面板中部 ✅ | P1 |
| 活跃对象 | 左主区 ✅ | 左面板 ✅ | P1 |
| 快速上手 | 左主区顶部 ⚠️ | 左面板底部 ✅ | P2 |

**关键发现**：

- **方案 A 问题**：快速上手模块放在主内容区顶部、活跃对象之上，占据了黄金位置。对已完成 3/5 步的用户来说，这个模块应该下沉或可折叠。
- **方案 B 优势**：快速上手放在左面板底部，信息优先级处理更合理。
- **两者共同缺失**：KPI 区目前没有展示"Token 用量/费用"指标。产品要求的"今日/本周/按角色分层统计"在 4 个 KPI 卡里没有位置。建议将"本周完成率"替换为"Token 用量"或增加第 5 个 KPI 卡。

## 3. 扩展性评估

### 未来确定要加入的模块（二期规划）

- 角色原型池详情页
- Token 用量趋势图
- 告警阈值配置
- 环境诊断面板
- 跨项目视图（三期）

### 方案 A 扩展性：⭐⭐⭐⭐（4/5）

- **左侧栏导航**天然支持新增页面，加一个 nav item 即可。
- **2:1 布局（主区 + 右栏）**在增加更多卡片时，主区可以垂直堆叠，结构不会崩塌。
- 风险：右栏 380px 固定宽度在大屏上可能显得紧凑，需要考虑 ≥1440px 时适当放宽。

### 方案 B 扩展性：⭐⭐⭐（3/5）

- **顶部 pill tabs 导航**在页面增多时会横向拥挤，6 个 tab 已经接近上限。
- **1:1 等宽双面板**在内容不对称时（比如左边只有一个表格、右边堆了阻塞+进度+日志三个模块）会造成一侧空白、一侧拥挤。当前设计中右面板已经比左面板内容密度高很多。
- 优势：全宽布局在数据密度增大时有更大的水平空间。

**结论**：方案 A 的侧栏导航 + 非等宽布局在中期扩展上更稳。

---

## 4. 明确推荐

### ✅ 推荐方案 A（暖色结构化），附 3 项改进建议

**核心理由**：

1. **场景匹配**：一期用户是项目发起人做日常浏览，不是 SRE 盯监控屏。暖白背景 + 结构化卡片的阅读舒适度更高，视觉疲劳更低。
2. **扩展性更优**：侧栏导航 + 2:1 非等宽布局在二期加模块时不需要改架构。
3. **无障碍基础更扎实**：亮色背景下的色彩对比度更容易达标，不需要在深色底上反复调色。
4. **与产品定位一致**：产品定义是"项目控制台原型"而非"运维监控系统"，暖色结构化风格传达的心智模型更准确。

### 必须执行的改进项

| # | 改进 | 理由 | 优先级 |
|---|------|------|--------|
| 1 | **快速上手模块下移或增加折叠**：当步骤完成过半后自动收起，将黄金位置让给活跃对象 | 当前占位不符合信息优先级 | P0 |
| 2 | **KPI 区增加 Token 用量卡**：替换"本周完成率"或扩为 5 卡布局 | 产品 P0 要求的 token 统计在当前 KPI 中缺失 | P0 |
| 3 | **吸收方案 B 三个交互细节**：① sparkline 迷你趋势图加入 KPI 卡；② 顶栏搜索框 + `/` 快捷键；③ 系统在线心跳指示灯 | 这些细节提升信息密度和操作效率，与方案 A 风格不冲突 | P1 |

## 5. 补充说明

- 方案 B 不建议废弃，可作为**二期"深色模式"切换选项**沉淀。当产品演进到多项目并行、需要高密度数据展示时，指挥中心风格会重新适用。
- 两套方案在无障碍、响应式、设计规范遵循上都做得很好，Lux 的工作质量没有问题。
- 产品验收时，建议优先用场景二（"李琪 10 秒内看到核心状态"）做走查测试。
