# UI/体验复查收口清单
整理人：加里奥
时间：2026-03-19
来源：拉克丝（视觉）+ 杰斯（办公视角）

## 高严重级
| 问题现象 | 严重级别 | 建议处理方式 | 本轮修复 |
|---------|---------|------------|---------|
| CSS 加载顺序与设计意图相反，tokens.css 覆盖了 style.css 中的设计 token，导致设计系统覆盖逻辑失效 | 高 | 调整 index.html 中样式引入顺序，统一为 tokens.css → style.css，并回归检查全站样式是否恢复预期 | 是 |
| 目录中遗留旧版 styles.css，与 style.css / tokens.css 存在主色、圆角、侧栏宽度等冲突定义，存在被误引用后导致整体视觉漂移的风险 | 高 | 清理或下线旧版 styles.css，统一单一样式入口，并全局排查引用来源避免混用 | 是 |
| 状态色尤其错误色在多个文件中定义不一致，跨组件会出现色差，破坏状态表达一致性 | 高 | 收敛状态色 token 到唯一来源，统一错误/成功/警告语义色，并批量替换组件引用 | 是 |
| 控制台当前整体为只读模式，用户无法新建任务、下指令、备注进度，无法形成真实办公闭环 | 高 | 明确本期最小可操作闭环，至少补齐任务创建、指派/触发、进度备注等核心动作入口 | 是 |
| 任务与 KPI 数据使用静态演示数据，页面又未明确提示非实时，容易让用户误判真实状态 | 高 | 尽快接入真实数据源；若本期无法接通，需显著标注“演示/静态数据”，避免业务误用 | 是 |
| 导航中的“记忆”“文档”等入口使用开发者语言，普通办公用户无法理解用途与价值，核心信息架构偏离办公场景 | 高 | 将导航与页面命名改成办公语义，并补充用途说明；必要时重组信息架构，弱化内部系统概念暴露 | 是 |

## 中严重级
| 问题现象 | 严重级别 | 建议处理方式 | 本轮修复 |
|---------|---------|------------|---------|
| 大卡片容器圆角跨度过大（12/16/20px 混用），同层级模块视觉层次混乱 | 中 | 建立页面级容器圆角规范，统一大卡片与特殊卡片的 token 使用策略 | 是 |
| Inspector Sidebar 使用 fixed 脱离 grid 文档流，导致主内容右侧保留空白，可用宽度被浪费 | 中 | 调整侧栏布局实现，使其与 shell grid 对齐；同步检查展开/收起态的宽度占位逻辑 | 是 |
| 多类卡片 padding 使用大量非 token 且非对称数值，页面节奏不统一 | 中 | 梳理卡片内边距规范，优先替换为现有 spacing token，减少临时值 | 是 |
| 活跃导航项通过 border-left + padding 计算补偿，文字存在轻微左移抖动风险 | 中 | 改为不会影响文本起始位的激活态方案，如伪元素高亮、背景条或统一占位 | 是 |
| Topbar 右侧状态区信息堆叠过多，窄窗口会折行并拉高顶栏，重要状态被噪音淹没 | 中 | 重构顶栏信息层级，保留核心系统状态与主操作，将次要状态折叠或下沉到详情区 | 是 |
| “强制重新加载”按钮表述过于技术化且带压迫感，不适合作为办公默认刷新动作 | 中 | 改名为“刷新”或“同步”，将强制刷新作为次级/异常场景操作并补充说明 | 是 |
| 搜索快捷键仅显示 ⌘K，对 Windows/Linux 用户不友好 | 中 | 改为 Ctrl/⌘K 双提示或按平台动态显示快捷键文案 | 是 |
| 任务页同时存在筛选条、列表/看板切换及隐藏的第三套 board-view 结构，视图关系不清晰 | 中 | 清理废弃视图结构，保留一套清晰的信息与切换逻辑，并强化当前视图状态反馈 | 是 |
| KPI 仅显示当前值，没有趋势或环比信息，用户难以判断好坏与变化方向 | 中 | 为关键指标补充趋势、环比或基线说明，至少提供较昨日/较上周变化 | 是 |
| Inspector Sidebar 折叠后入口过小且缺乏关联提示，用户难以发现其中的待处理信息 | 中 | 放大入口并与主内容建立视觉锚点，增加“待处理事项”等显性提示 | 是 |
| 设置页内容几乎全是系统管理员视角，普通办公用户找不到与自己相关的设置项 | 中 | 区分“系统设置”和“个人/团队设置”，本期至少补齐通知偏好、语言、时区等常用项或隐藏无关项 | 待定 |
| Agent 状态页缺少可执行动作与上下文，用户只能看状态，不能发起管理操作 | 中 | 为 Agent 页补充查看任务、下达指令、暂停/重试/分配等基础操作入口，若本期范围过大至少补查看上下文 | 待定 |

## 低严重级
| 问题现象 | 严重级别 | 建议处理方式 | 本轮修复 |
|---------|---------|------------|---------|
| style.css 中 @keyframes spin 重复定义，虽不影响功能但增加维护噪音 | 低 | 删除重复定义并顺手做一次样式文件去冗余清理 | 待定 |
| 多处关键字号使用硬编码值，未纳入字体 token 体系，后续整体调字号时难统一联动 | 低 | 为大数字与特殊标题补齐字号 token，并逐步替换硬编码值 | 待定 |
| --ease-fast 命名语义与实际值类型不一致，变量名像 easing curve，但存的是完整 transition shorthand | 低 | 统一 motion token 命名语义，区分 duration/easing/transition 级别 | 待定 |
| .segment-item、.pie-legend-dot 等小组件圆角存在硬编码，和 token 系统脱节 | 低 | 收敛小圆角值到统一 token，减少零散硬编码 | 待定 |
| --text-active 属于 alias of alias，增加理解成本 | 低 | 清理多余别名，保留必要的语义映射层 | 待定 |
| 总览页“加载中…”缺少超时与失败提示，用户难判断是慢还是出错 | 低 | 增加加载超时、失败回退与重试提示文案 | 待定 |
| 导航图标全部使用 emoji，跨系统渲染差异大，专业工具感不足 | 低 | 逐步替换为统一图标体系，保证跨平台风格一致 | 待定 |
| 协作页使用“Subagent”等内部术语，普通办公用户难理解 | 低 | 将页面标题和字段转成办公语言，保留技术词作为辅助说明而非主标题 | 待定 |
| 文档页初始空态引导不足，用户首次进入时容易看到“两列空白” | 低 | 增加空态说明、示例内容或引导操作入口 | 待定 |
| 用量页时间段切换后缺少明显反馈，用户不确定切换是否生效 | 低 | 补充加载态、过渡反馈或选中态强化 | 待定 |
