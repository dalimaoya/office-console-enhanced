# 架构论证·视觉设计视角
**作者**：拉克丝（ui-lux）  
**日期**：2026-03-19  
**性质**：战略论证，不修改任何代码  
**论证角度**：视觉设计语言与叙事体系

---

## 一、两套故事线的设计语言潜力对比

### 英雄联盟角色体系

**能建立的设计语言**：
- **强辨识度角色视觉**：每个 Agent 有对应的英雄形象、专属色彩（如提莫的绿蘑菇绿、拉克丝的金粉光效、贾克斯的紫橙战意），可以为每个角色打造独立的徽章、图标、状态标识。
- **游戏感 UI 语言**：技能图标样式的操作按钮、血条/经验条感的进度指示器、符文/水晶质感的装饰元素、十六边形的卡片框等——有成熟的游戏 UI 设计范式可借鉴。
- **叙事情感调度**：角色有个性，有台词，有"出场动画"的仪式感，适合制造"召唤师下令，英雄响应"的执行感。

**最大挑战**：  
英雄联盟的视觉语言本质是 **"对抗场"语言**——高对比度、高饱和、强节奏。每个英雄的视觉基因是独立且强烈的，难以统一到一套安静、连贯的界面设计系统里。想象一下提莫的毒蘑菇绿和拉克丝的圣光金同框——不是不好看，但很难"平静"。

---

### 三省六部体系

**能建立的设计语言**：
- **汉式礼制美学**：朱墨双色体系、篆刻印章感的图标、宣纸质感的卡片、楷体/宋体的排版系统、折叠如奏折的面板展开动效。
- **层级感极强**：司礼监/六部的权级感本身就是视觉层级的叙事来源，可以用尺寸、位置、色泽的差异将层级可视化。
- **统一性更高**：汉制美学有统一的颜色体系（朱/金/墨/青），图形语言也有明确的历史参照，设计师更容易做到"一眼认出整套系统"。

**最大挑战**：  
汉制美学如果处理不精细，极容易滑向"廉价国风 App"的老套感。它需要极高的设计细节投入，否则质感会崩掉。

---

### 哪种更容易形成统一的视觉风格？

**结论：三省六部体系更容易形成统一的视觉风格。**

原因：汉制美学有高度聚合的颜色基因（金/朱/墨）和固定的装饰语言（印章/折叠/卷轴），设计约束更清晰，设计师更容易在这个框架里做一致性决策。

英雄联盟体系则需要为每个角色分别设计视觉子语言，然后再上一套统一框架来压制碎片化——执行难度更高，视觉统一性更脆。

## 二、"深度办公增强"应有的视觉气质

**核心气质关键词**：专注、精确、持续、沉静、信赖。

一个真正的"深度办公"工具，视觉上不应制造兴奋感和刺激感，而应传递：
- **我在这里，我稳定运转**
- **我帮你掌控全局，不是表演给你看**
- **每一个操作都被准确记录，每一个任务都在流动中**

这种气质接近的设计参照：Notion 暗色模式 + Linear + Vercel Dashboard——低调、精确、有密度感的信息展示。

---

### 英雄联盟的游戏感是否与之冲突？

**有明显冲突，但可以被管理。**

游戏感的核心问题在于：  
1. **英雄联盟 UI 是为"20分钟内快速决策"设计的**，它的视觉语言天然偏向即时反馈、高刺激、短周期——这和"深度持续执行"的产品节奏相反。
2. **角色的存在感会抢夺用户注意力**：用户的目光应聚焦在任务对象和工作内容上，而不是在"拉克丝今天穿什么皮肤"上。
3. **外部展示时存在认知割裂**：向同事或上级展示工作进展，突然弹出一个英雄联盟界面，解释成本很高。

**可以被管理的部分**：  
如果英雄联盟的元素只作为"角色徽章/头像"使用，不进入主界面的交互流，游戏感是可以被收敛的。关键是：**叙事在对话里，设计在系统里**——英雄的名字和个性出现在消息文本里，但控制台本身保持中性、专业。

## 三、军队体系 & 项目管理体系的设计机会

### 军队体系（战区/师旅/连）

**设计语言方向**：
- **作战地图感**：深绿/墨灰底色，卫星图纹理，任务以"战区标记"形式分布
- **命令链视觉化**：从上级到下级有明确的信号线/命令流动动效，类似战术态势图
- **进度用"占领/推进"表达**：任务完成度 = 阵地推进距离
- **身份标识**：军衔徽章、部队番号——每个角色有对应的军衔和作战单位

**设计机会**：  
军队体系的最大优势是"命令链的视觉本能"——每个人看到树状层级都能瞬间理解"谁指挥谁"。这对于架构可视化极有价值。它也是所有故事线里**最容易做层级感**的选项。

**风险**：  
军队感过重会产生压迫感，对女性用户和非军事文化背景用户存在排斥风险。

---

### 项目管理体系（PMO/PM/执行）

**设计语言方向**：
- **现代 Dashboard 感**：与 Linear、Notion、Jira 的视觉语言拉近距离，降低认知摩擦
- **泳道/看板/甘特混合表达**：任务状态用泳道视觉表达，层级关系用树状卡片组表达
- **无个性化角色**：角色用职能图标代替（产品经理图标 vs 开发图标），降低叙事负担
- **颜色系统**：可以采用专业工具常见的功能色（蓝=进行中，绿=完成，橙=阻塞，红=紧急）

**设计机会**：  
项目管理体系的最大优势是**零认知成本**——每一个用过项目管理工具的用户看到这套设计语言，立刻就能上手，不需要学习新的隐喻体系。它是最容易被团队接受、最容易演示给外部利益相关方的选项。

## 四、暗色叙事型设计底座（深蓝黑+金色）的故事线适配性

### 当前设计底座评估

深蓝黑 + 金色的暗色叙事型设计，本质上是**"权威感+仪式感"的视觉语言**，它的气质接近：
- 高端游戏控制台（LoL Esports 赛事大屏）
- 奢侈品品牌官网
- 企业级数据可视化大屏

这套设计语言**对不同故事线的适配性**：

| 故事线 | 适配度 | 原因 |
|--------|--------|------|
| 英雄联盟 | ★★★★★ | 天然契合，LoL 自己就是这套视觉语言 |
| 三省六部 | ★★★★☆ | 金色完美匹配，深蓝黑需要替换为墨黑或深赤 |
| 军队体系 | ★★★☆☆ | 金色在军队语境略显华丽，需要替换为军绿/作战灰 |
| 项目管理 | ★★☆☆☆ | 暗色+金色太有"仪式感"，与日常工作场景气质偏离 |
| 无故事线/通用 | ★★★★☆ | 去掉强符号装饰后，深色+强调色是专业工具的主流 |

---

### 如何做到"故事线无关"的设计底座

**核心原则：叙事层与视觉底层分离。**

具体方案：

**Layer 0 - 设计 Token 层（故事线无关）**
- 基础色板：`--color-bg-primary`, `--color-accent`, `--color-text-primary`
- 布局系统：模块间距、卡片尺寸、字号层级
- 交互语言：Hover 状态、Loading 状态、Empty State
- 这一层对任何故事线保持稳定，不携带任何叙事符号

**Layer 1 - 主题层（故事线绑定）**
- 颜色 Override：LoL 主题用深蓝金，古风主题用墨赤金，军队主题用深绿灰
- 图标系统 Override：每套故事线换一套图标语言
- 角色徽章/头像：叙事符号集中在此层
- 文字/措辞 Override：按钮文案、状态文案按故事线替换（"召唤" vs "奏请" vs "下令"）

**Layer 2 - 叙事增强层（可选开关）**
- 进入/退出动效的叙事化处理（英雄出场 vs 圣旨展开）
- 装饰边框和分隔线的风格化
- 空状态和错误状态的叙事化文案

**实现方式**：CSS 变量 Token 系统 + 主题 Class 切换，前端通过加载不同主题包实现故事线切换，不影响核心组件逻辑。

这样就能做到：**换故事线 = 换主题包，不重写界面**。

## 五、推荐故事线框架

### 🏆 推荐：混合体系 —— "英雄联盟叙事 + 项目管理逻辑 + 暗色底座"

**推荐理由**：

**1. 叙事层保留英雄联盟**  
英雄联盟角色体系的最大价值不是设计语言，而是**人格化**——它让每个 Agent 变得有辨识度、有个性、有记忆点。"提莫在帮我推进任务"比"总控 Agent 在执行调度"更有意思，团队成员也更容易记住"找拉克丝做设计"而不是"找 UI Agent"。这个价值不应该丢掉。

**2. 底层逻辑切换到项目管理体系**  
叙事层的英雄联盟不应该渗入架构层级命名。底层的层级关系应该用项目管理语言描述（全局总控/项目指挥官/领域 Agent/执行 Agent），这样架构可以独立于故事线存在，未来换故事线不需要重写文档。

**3. 视觉底座保持故事线可替换**  
按上一节的方案，做分层 Token 系统，当前默认加载 LoL 主题，但保留接口可以切换为古风、军队或通用专业主题。

**4. 这是最能兼容"个人使用"和"对外展示"两种场景的方案**  
- 个人使用时：英雄联盟的叙事感让日常工作更有趣，降低使用疲劳
- 对外展示时：切换到通用专业主题，深色 + 简洁图标，完全可以展示给团队或甲方

---

### 备选：三省六部体系（如果要强调"中文软件原创性"）

如果产品定位是"面向中文用户的原创工作流工具"且有意做对外传播/开源社区曝光，三省六部体系的叙事话题性和国风美学独特性会带来更高的记忆点。但执行门槛高，需要专门的设计资源投入才能避免廉价感。

## 六、对"当皇上"开源项目的视觉评价

从设计角度看，"当皇上"是一个**叙事优先、界面次之**的项目。它的价值在于叙事框架本身带来的话题性和记忆点，而不在于产品级的 UI 精细度。

我们和它的核心差异在于：**我们想做一个真实用于深度工作的工具，而不只是一个有趣的概念演示**。这个差异决定了我们的视觉选择必须以"可持续使用"为第一优先级，而不是以"一眼惊艳"为优先。

---

## 七、设计行动建议

1. **立即可做**：整理当前控制台 UI 的设计 Token，分离叙事符号层和功能层
2. **短期**：为 LoL 主题建立一套正式的角色徽章设计规范（每个 Agent 一枚徽章，统一圆形/六边形规格）
3. **中期**：建立主题切换机制，预留"通用专业主题"的切换入口
4. **长期**：如果产品走向对外开放，可以为不同用户群体提供主题包（LoL 主题/古风主题/极简专业主题）
