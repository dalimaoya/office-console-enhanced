# 办公控制台增强项目

## 当前状态

这个项目已清空旧需求、旧方案、旧任务和旧代码相关内容。

当前目录仅保留一个全新的项目骨架，用于重新梳理需求并重新启动项目。

## 现阶段目标

- 重新定义项目目标
- 重新梳理需求范围
- 重新确认角色分工
- 重新建立任务、决策与交付结构

## 当前目录用途

- `briefs/`：任务简报和输入对象
- `docs/`：新需求、新方案、新说明
- `tasks/`：新任务拆解与状态跟踪
- `decisions/`：新的关键决策记录
- `protocols/`：新的项目协议
- `reviews/`：评审输入与评审结论
- `status/`：项目状态记录
- `contracts/`：项目约束、接口约定、角色交接约定
- `artifacts/`：新产物输出
- `handoffs/`：正式交接材料目录
- `notifications/`：仅用于通知留痕，不作为正式交接依据

## 规则

- 不继承旧项目资料作为默认执行依据
- 如需回看旧内容，只能从备份中人工查阅
- 新一轮需求梳理完成前，不默认沿用旧假设
- 项目正式接力走 Teemo 内部调度
- 正式交接走成果物，不走群消息
- 群/聊天消息只做进度可见，不作为正式交接依据
- Teemo 是该项目唯一项目指挥官
- specialist 默认对 Teemo 负责
- `handoffs/` 是唯一正式交接目录；`handoff/` 仅保留兼容用途，不再作为新的正式交接落点
