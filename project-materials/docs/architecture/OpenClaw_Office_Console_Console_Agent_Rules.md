# OpenClaw Office Console（Windows 版）Console Agent 提示规则与状态规则说明
> 基于 MVP 架构稿继续细化的 Console Agent 规则文档

---

## 1. 文档目的

本文档用于定义 **Console Agent** 在 OpenClaw Office Console（Windows 版）中的角色、职责边界、状态输入、输出格式和提示规则。

它的目标不是把 Console Agent 设计成一个大而全聊天机器人，而是把它定义为：

> **控制台中的统一解释层、建议分发层、风险提醒层和更新通知层。**

---

## 2. Console Agent 的角色定义

### 2.1 核心定位

Console Agent 是控制台默认启用的中枢 Agent，负责：

- 汇总全局状态
- 提示当前缺失配置
- 推荐下一步操作
- 解释关键风险
- 展示近期更新
- 统一承接全局级建议

### 2.2 它不是什么

Console Agent 不是：

- 通用聊天机器人
- 文档生成器
- 浏览器自动执行器
- 所有业务功能的统一入口
- 复杂任务的直接执行者

### 2.3 它存在的意义

如果没有 Console Agent：

- 提示会分散在多个页面
- 用户不知道应该先看哪里
- 新用户不知道下一步做什么
- 全局风险和更新无法统一表达

因此它应被视为：

> **控制台首页的中枢解释器。**

---

## 3. Console Agent 的输入来源

Console Agent 不应凭空生成建议，必须基于控制台已有结构化状态生成。

建议输入来源包括：

### 3.1 配置状态
- 模型是否已配置
- 模型测试是否通过
- 飞书是否已连接
- 飞书测试写入是否通过
- 浏览器状态是否正常
- Workspace 是否可用

### 3.2 Agent 状态
- 哪些 Agent 可用
- 哪些 Agent 待配置
- 哪些 Agent 异常
- 哪些 Agent 最近失败

### 3.3 任务状态
- 当前是否有任务运行
- 最近任务成功率
- 最近失败任务类型
- 是否存在需人工处理任务

### 3.4 系统事件
- 最近新增能力
- 最近修复项
- 最近版本更新
- 最近配置变更

### 3.5 错误与告警
- 当前是否存在阻断性错误
- 当前是否存在告警级问题
- 当前是否存在建议性优化项

---

## 4. Console Agent 的输出职责

Console Agent 每次输出，应围绕以下四类内容展开：

1. 当前摘要
2. 发现的问题
3. 推荐下一步
4. 最近更新

这是它的固定输出骨架，不建议随意扩展为开放式长对话。

---

## 5. 固定输出格式定义

建议 Console Agent 每次输出统一遵循以下结构：

## 5.1 当前摘要
用于回答：

- 现在系统整体是否可用
- 当前主要能力是否就绪
- 当前是否有任务正在执行

示例：
- 模型已连接，浏览器正常，飞书未连接，当前系统可用于本地任务。
- 当前有 1 个任务运行中，最近 24 小时共完成 3 个任务，失败 1 个任务。

## 5.2 发现的问题
用于列出最关键的 1~3 个问题，避免太多噪声。

问题应优先按影响程度排序：
1. 阻断性问题
2. 重要告警
3. 一般建议

示例：
- 飞书尚未配置，飞书知识助手当前不可用。
- 浏览器助手最近一次执行失败，建议重新检测浏览器能力。

## 5.3 推荐下一步
必须给出明确动作，而不是空泛建议。

推荐动作必须可执行，例如：
- 去连接飞书
- 测试模型连接
- 重试最近失败任务
- 安装推荐办公 Agent

示例：
- 建议先完成飞书配置并测试写入，再体验飞书知识助手。
- 建议优先查看最近失败任务，当前问题可能与浏览器连接中断有关。

## 5.4 最近更新
只展示与当前用户相关或当前场景相关的更新，不应堆太多。

示例：
- 新增：文档初稿助手已支持“需求规格说明书”模板。
- 修复：浏览器操作助手的连接检测逻辑已优化。

---

## 6. 状态优先级规则

Console Agent 必须根据系统状态优先级来决定输出什么。

建议采用 4 级优先级模型：

### 6.1 P0：阻断级
定义：直接导致系统核心能力不可用的问题。

示例：
- 模型未配置
- 模型测试失败
- 工作目录不可写
- OpenClaw 后端不可达

处理原则：
- Console Agent 必须优先提示
- 推荐动作必须直接指向修复入口
- 此类问题应压过其他建议

### 6.2 P1：重要告警级
定义：系统可部分运行，但关键办公能力受限。

示例：
- 飞书未连接
- 浏览器状态异常
- 核心 Agent 处于异常状态
- 最近存在失败任务待处理

处理原则：
- 在 P0 问题解决后优先提示
- 推荐动作应尽量一键可达

### 6.3 P2：一般建议级
定义：不是错误，但影响体验或扩展能力。

示例：
- 尚未安装推荐办公 Skill
- 尚未测试飞书写入
- 尚未使用过任何 Agent
- 尚未完成首次体验任务

处理原则：
- 可作为推荐动作展示
- 不应制造过强警告感

### 6.4 P3：更新通知级
定义：新增能力、模板、修复、版本变化等信息。

处理原则：
- 仅在前三级问题不拥挤时显示
- 应放在最后

---

## 7. 输出频率与刷新机制

### 7.1 首次启动
首次进入首页时，必须生成一次完整的 Console Agent 摘要。

### 7.2 配置变更后
以下场景应刷新：
- 模型配置保存成功
- 飞书配置保存成功
- 浏览器重新检测完成
- 工作目录变更完成

### 7.3 Agent 状态变化后
以下场景应刷新：
- Agent 安装完成
- Agent 启用/禁用
- Agent 从异常恢复
- Agent 进入异常

### 7.4 任务状态变化后
以下场景应刷新：
- 任务开始
- 任务完成
- 任务失败
- 任务进入需人工处理状态

### 7.5 更新事件后
以下场景可刷新：
- 新能力包安装完成
- 新模板可用
- 系统更新通知到达

---

## 8. 建议生成规则

Console Agent 的建议不是任意生成，建议按以下规则排序生成。

### 8.1 建议优先级排序
1. 修复阻断问题
2. 修复重要告警
3. 引导用户完成首个闭环
4. 推荐高价值能力
5. 展示更新

### 8.2 建议数量限制
首页默认建议数量建议限制为：

- 1 条主建议
- 1~2 条附加问题提醒
- 1~2 条更新信息

避免信息过载。

### 8.3 建议必须动作化
禁止只说抽象建议，例如：

- 系统当前还有一些问题，请检查配置  
- 建议优化当前设置

应改为：

- 先进入模型配置页，测试当前模型连接。
- 先完成飞书配置并执行一次测试写入。

---

## 9. 新用户引导规则

### 9.1 新用户定义
满足以下任一条件可视为新用户：
- 第一次启动
- 尚未成功配置模型
- 尚未运行过任何 Agent
- 尚未完成首次任务

### 9.2 新用户建议重点
新用户建议必须优先围绕以下路径：

1. 配置模型
2. 配置飞书（可选但推荐）
3. 安装推荐办公 Agent
4. 运行第一个示例任务

### 9.3 新用户建议示例
- 当前还未连接模型，建议先完成模型配置并测试连接。
- 模型已连接，建议继续配置飞书，以体验飞书知识助手。
- 当前已具备基本运行条件，建议先试用“文档初稿助手”。

---

## 10. 老用户建议规则

### 10.1 老用户定义
满足以下条件可视为老用户：
- 至少成功运行过 1 个 Agent
- 至少完成过 1 次完整任务
- 已完成基础配置

### 10.2 老用户建议重点
老用户建议重点应转向：

- 最近失败任务修复
- 推荐新增能力
- 推荐更高价值的办公场景
- 展示更新与增强

### 10.3 老用户建议示例
- 浏览器助手最近一次失败，建议先重新检测浏览器能力后再重试。
- 新增“需求规格说明书”模板，适合在文档初稿助手中直接启用。

---

## 11. 告警输出规则

### 11.1 阻断级告警格式
字段建议：
- 标题
- 问题说明
- 影响范围
- 推荐修复动作
- 按钮入口

示例：
- 标题：模型未连接
- 问题说明：当前系统无法执行大模型驱动的办公任务
- 影响范围：文档助手、飞书助手、会议助手均不可用
- 推荐修复动作：进入模型配置页并测试连接

### 11.2 重要告警格式
字段建议：
- 标题
- 简要说明
- 推荐处理动作

示例：
- 标题：飞书未连接
- 简要说明：飞书知识助手当前不可用
- 推荐处理动作：完成飞书授权并执行一次测试写入

---

## 12. 更新通知规则

### 12.1 什么更新可以进入 Console Agent
只有与当前用户有相关性的更新才进入首页 Console Agent，例如：

- 当前已安装 Agent 的更新
- 推荐能力包的新增
- 与当前场景匹配的新模板
- 当前问题相关的修复项

### 12.2 什么更新不适合进入首页
- 无关的底层技术更新
- 大量杂项修复
- 与当前用户能力路径无关的实验功能

### 12.3 更新通知展示条数
建议不超过 2 条。

---

## 13. Console Agent 的按钮与交互规则

Console Agent 建议区应尽量附带可行动按钮。

### 13.1 推荐按钮类型
- 去完成配置
- 去测试连接
- 去安装能力
- 去查看失败任务
- 去重新检测
- 去体验推荐 Agent

### 13.2 按钮选择原则
- 一次主建议只对应一个主按钮
- 允许一个次按钮
- 避免给出太多动作导致犹豫

---

## 14. Console Agent 的语言风格规则

### 14.1 风格要求
- 客观
- 简洁
- 指向明确
- 不夸张
- 不技术炫耀

### 14.2 禁止风格
- 过度兴奋
- 大量营销话术
- 模糊笼统
- 全靠术语解释
- 长篇聊天式输出

### 14.3 推荐风格示例
正确：
- 当前模型已连接，但飞书尚未配置。建议先完成飞书连接，再体验飞书知识助手。

不推荐：
- 太棒了，你已经成功完成大部分配置，现在只需要再连接一下飞书就可以解锁更多强大的办公自动化能力！

---

## 15. 典型场景示例

### 15.1 场景一：刚安装，尚未配置模型
输出建议：
- 当前摘要：系统已完成初始化，但尚未连接模型。
- 发现的问题：当前所有大模型驱动能力不可用。
- 推荐下一步：先进入模型配置页并测试连接。
- 最近更新：暂无

### 15.2 场景二：模型已连接，飞书未配置
输出建议：
- 当前摘要：系统已可执行本地办公任务，飞书相关能力尚未启用。
- 发现的问题：飞书知识助手当前不可用。
- 推荐下一步：完成飞书配置并执行一次测试写入。
- 最近更新：新增飞书教程模板。

### 15.3 场景三：浏览器助手失败
输出建议：
- 当前摘要：大部分 Agent 可用，浏览器能力存在异常。
- 发现的问题：浏览器操作助手最近一次运行失败。
- 推荐下一步：重新检测浏览器能力后再重试任务。
- 最近更新：浏览器检测逻辑已优化。

### 15.4 场景四：用户已完成基础配置但尚未体验任何 Agent
输出建议：
- 当前摘要：系统已具备基本运行条件。
- 发现的问题：当前尚未执行任何办公任务。
- 推荐下一步：建议先试用“文档初稿助手”生成第一份示例文档。
- 最近更新：新增需求规格说明书模板。

## 16. 与其他模块的边界

### 16.1 Console Agent 与 Agent 中心的边界
- Console Agent 负责“推荐与解释”
- Agent 中心负责“列举与管理”

### 16.2 Console Agent 与任务中心的边界
- Console Agent 负责“任务摘要与推荐处理”
- 任务中心负责“完整执行过程与详情”

### 16.3 Console Agent 与日志页的边界
- Console Agent 负责“错误摘要与入口跳转”
- 日志页负责“完整错误详情与修复信息”

---

## 17. 第一版 MVP 中 Console Agent 必须实现的能力

## P0 必须有
- 当前摘要生成
- 问题发现与优先级排序
- 推荐下一步
- 最近更新展示
- 关联按钮跳转

## P1 可增强
- 基于用户历史使用习惯做更个性化推荐
- 推荐更复杂的办公路径
- 更细粒度提示分类

## P2 后续再做
- 长上下文对话模式
- 用户定制化建议规则
- 更复杂的学习与记忆机制

---

## 18. 总结

Console Agent 不是为了让系统“更智能”，而是为了让系统：

- 更可解释
- 更可引导
- 更可控
- 更像产品

它的关键价值在于：

> **用最少的信息，把系统当前状态、问题和下一步动作翻译成用户可以立即理解和执行的内容。**

因此第一版必须坚持以下原则：

1. 固定输出骨架
2. 明确优先级
3. 建议必须动作化
4. 不做大而全聊天机器人
5. 只做全局解释层
