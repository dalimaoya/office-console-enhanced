## 二、我们的优势（CC 没有的独特能力）

### 8 项核心差异化优势

1. **SSE 实时推送全链路** - Gateway → SseHub → 浏览器实时透传，无需手动刷新（CC 依赖定时拉取）
2. **飞书 Webhook 告警通知** - agent 异常/任务阻塞自动推送飞书，办公生态深度集成
3. **YAML 模板系统** - 5个预置办公模板，一键应用标准化 agent 配置，降低启动门槛
4. **IP 白名单写操作控制** - 基于来源 IP 的安全边界细化，比 CC 的 token-only 更细粒度
5. **三路数据源可切换** - FileReader / GatewayWsClient / CliAdapter，容灾能力强
6. **开发体验优势** - tsx 零构建运行 + 热重载，开发迭代更快速
7. **Controller/Service/Data 三层架构** - 比 CC 的 runtime+ui 二层更清晰，扩展性更好
8. **任务看板可视化** - Kanban 式看板视图，比 CC 的纯列表更直观展示任务状态流转

### 办公场景专项优势

9. **实时倒计时与连接状态感知** - 显式展示刷新计时、缓存状态、连接健康度
10. **飞书生态协同** - 告警直接触达办公协作平台，符合中国办公习惯
11. **办公模板预设** - 开箱即用的办公场景 agent 配置方案

## 四、优先补齐建议

### P0（立即做 - 最影响专业用户使用）

1. **任务写操作 API** - PATCH /api/v1/tasks/:id/status 与 POST /api/v1/tasks
   - **影响**: 控制台不能推进任务状态 → 用户必须跳出控制台 → 中断工作流
   - **优先级**: 最高，使控制台从"观察"变为"工作"工具

2. **全站搜索系统** - GET /api/v1/search?q=&type= 与前端搜索框
   - **影响**: 10+ agent 场景下无法快速定位 → 滚列表浪费时间 → 专业用户无法忍受
   - **优先级**: 最高，搜索是专业工具的基础设施

3. **成本感知深化** - token 消耗归因 + 上下文压力指标
   - **影响**: 不知道谁在花钱、谁快满context → 成本失控风险
   - **优先级**: 高，成本控制是专业用户的核心关切

4. **Agent 状态精确化** - working/idle/backlog/blocked + 当前任务显示
   - **影响**: 无法区分"正在执行"vs"待命" → 排班调度困难
   - **优先级**: 高，管 AI 团队需要精确状态

5. **设计 Token 系统** - 间距/字号/圆角 token 化 + focus-visible
   - **影响**: 视觉不一致、键盘用户体验为零 → 专业产品感缺失
   - **优先级**: 高，一次性投入，长期收益大

### P1（下一迭代 - 重要但非紧急）

6. **待办/异常聚合面板** - /api/action-queue + 异常流分级展示
   - **价值**: 从"被动告警"到"主动管理"，首页可见待处理事项
   - **复杂度**: 中

7. **Cron 定时任务健康监控** - /api/v1/cron 端点
   - **价值**: 定时任务静默失败需要感知，自动化场景必备
   - **复杂度**: 低

8. **会话Gateway深度集成** - sessions_list/history/status API集成
   - **价值**: 更精确的会话数据，支撑会话详情页
   - **复杂度**: 中

9. **表单验证系统 + Toast通知** - 为写操作提供UI反馈
   - **价值**: 写操作需要完整的前端交互支撑
   - **复杂度**: 中

10. **接线状态诊断卡片** - /api/v1/settings/wiring-status
    - **价值**: 数据异常时自助排查，增强用户信任
    - **复杂度**: 低

### P2（可暂缓 - 锦上添花）

11. **审批操作系统** - 高风险操作审批流程
    - **价值**: 多人协作场景安全必备，但当前单人单团队场景非紧急
    - **复杂度**: 高

12. **导入/导出备份** - 状态快照备份恢复
    - **价值**: 生产环境容灾，但不影响日常使用
    - **复杂度**: 中

13. **任务回放索引** - 历史问题追溯
    - **价值**: 排障能力提升，但频率较低
    - **复杂度**: 高

14. **浅色主题 + 国际化** - 增强适用性
    - **价值**: 扩大用户范围，但当前中文团队够用
    - **复杂度**: 高

### P3（不做 - CC 有但我们不需要的）

15. **安全风险翻译卡片** - 非技术用户语言解释风险
    - **理由**: 专业用户能直接读懂配置，不需要"小白语言"解释

16. **Done Checklist 评分体系** - readiness 评分
    - **理由**: 专业用户一眼就知道接了什么，过度工程化

17. **详细安装向导** - INSTALL_PROMPT.md 等
    - **理由**: 专业用户自己会装，不需要控制台详细向导

18. **过度新手引导** - 控制台内嵌详细文档
    - **理由**: 专业用户读 README 够了，控制台应聚焦核心操作

## 五、建议路线图（3 个迭代）

### 迭代 1：让控制台"可用"（现在 → 30天）

**主题**：补齐核心写操作与搜索能力

**重点产出**：
1. **任务写操作完整实现**
   - 后端：POST/PATCH /api/v1/tasks 系列 API
   - 前端：任务卡片可交互，状态流转 UI
   - 测试：写操作安全守卫+测试覆盖率

2. **全站搜索系统**
   - 后端：统一搜索端点 /api/v1/search
   - 前端：顶栏搜索框 + Cmd+K 快捷键
   - 支持 agent/task/session 三种类型搜索

3. **设计 Token 系统升级**
   - CSS: 间距/字号/圆角 token 化（4级/7级/5级）
   - 前端: focus-visible 可及性补全
   - 文档: 设计 token 文档与约定

**成功标准**：
- 用户能在控制台里创建/推进任务
- 搜索任意 agent/task/session ≤ 3秒
- 视觉一致性提升 30%，键盘导航可用

### 迭代 2：让控制台"好用"（30-60天）

**主题**：深化数据洞察与运营能力

**重点产出**：
1. **成本感知深化**
   - token 消耗归因视图（按 agent/task 拆分）
   - 上下文压力指标（接近 context 上限预警）
   - 用量趋势图表（简单折线图）

2. **Agent状态精确化**
   - agent 状态细化：working/idle/blocked/backlog
   - 当前任务显示 + 最近产出展示
   - 排班状态感知（忙/闲/卡/等）

3. **待办/异常聚合**
   - 待办操作队列 API 与前端卡片
   - 异常流分级展示（严重/警告/信息）
   - Cron 健康监控面板

4. **运维体验提升**
   - .env 模板文件（配置外部化）
   - 接线状态诊断卡片
   - 基础 Toast 通知系统

**成功标准**：
- 成本分析支撑决策，token 消耗透明化
- agent 状态精度从 50% → 90%
- 首页能一眼看到"待处理事项"
- 新手配置时间减少 50%

### 迭代 3：让控制台"专业"（60-90天）

**主题**：进阶治理能力与差异化强化

**重点产出**：
1. **审批操作系统**
   - 高风险操作审批流程
   - 审批历史记录与审计
   - 审批策略可配置

2. **会话深度集成**
   - Gateway sessions API 完整集成
   - 会话详情页带双语支持
   - 会话关联图谱

3. **差异化优势深化**
   - SSE 实时性增强（连接保活优化）
   - 飞书集成扩展（更多告警场景）
   - YAML 模板生态（模板市场雏形）

4. **专业运维能力**
   - 导入/导出备份
   - 任务回放索引
   - PM2 生产部署支持

**成功标准**：
- 多人协作场景安全完备
- 会话管理能力比肩 Gateway
- 差异化优势持续领先
- 生产环境下稳定运行 30天

## 六、一句话结论

**当前控制台是"高品质观察工具"，但专业用户需要的是"可以在里面工作的工具"；核心缺口不在功能数量，在于"能不能做事"，首要补齐任务写操作和搜索能力让用户不再需要跳出控制台。**
