# PC 办公控制台 UI/UX 产品规范
**作者**: 产品经理艾克（product-ekko）  
**日期**: 2026-03-19  
**触发原因**: 用户李琪反馈控制台配色糟糕、PC 界面呈现手机比例、不支持自适应  
**下游消费者**: 设计师拉克丝（ui-lux）、前端伊泽瑞尔（frontend-ezreal）

---

## 1. PC 办公控制台定位说明

### 目标用户
- **主用户**：有 AI 工程能力、长期使用 OpenClaw 的专业用户（工程师/产品经理）
- **使用频次**：每日多次打开、长时间停留（非偶发访问）
- **操作模式**：键盘+鼠标为主，需要多区域同时可见，高频切换

### 使用场景
- 监控多个 Agent 状态（不关闭、挂在后台实时更新）
- 查看任务队列、处理告警、快速搜索（Cmd+K）
- 审阅文档、查看 Memory、调整 Settings

### 设备环境（强制约束）
| 参数 | 约束值 | 说明 |
|------|--------|------|
| 主战场分辨率 | 1440px ~ 2560px | MacBook Pro 外接显示器为主 |
| 最低支持分辨率 | 1280px | 低于此宽度不做保证 |
| 最低支持高度 | 720px | 正常笔记本屏幕 |
| 浏览器 | Chrome / Safari 最新版 | 不要求兼容 IE |
| **移动端** | **不支持** | 本阶段明确不面向手机场景 |

## 2. 页面比例与布局规范（强制规则）

### 2.1 整体布局结构

```
┌─────────────────────────────────────────────────────────┐
│  侧边栏(240px)  │          主内容区(1fr)          │ 检查器(260px) │
│  固定宽度       │          自适应撑满             │ 可折叠         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 强制宽度约束

| 元素 | 规范值 | 类型 |
|------|--------|------|
| `body` / `.shell` 最小宽度 | **1280px** | **强制，不可省略** |
| 侧边栏（`.sidebar`）宽度 | **240px** | 固定，不随窗口缩小 |
| 检查器侧边栏（`.inspector-sidebar`）宽度 | **260px** | 可折叠，折叠后消失 |
| 主内容区（`.main`）最小宽度 | **640px** | 防止内容区过窄 |
| 主内容区（`.main`）内边距 | **24px 32px** | 上下24px / 左右32px |
| 内容区最大内容宽度 | **无限制**（充分利用空间） | 不加 `max-width` 居中限制 |

**关键修复点**：当前代码 `body` 和 `.shell` 均未设置 `min-width`，导致在窄窗口（或某些浏览器缩放比例下）自动压缩成手机比例。**必须在 body 和 .shell 均加 `min-width: 1280px`**。

### 2.3 侧边栏规范

- 侧边栏固定在左侧，全高度（`height: 100vh`，`position: sticky; top: 0`）
- 导航项高度：44px
- 导航项左侧内边距：16px
- 图标 + 文字标签，两者均在 1280px+ 时可见（**禁止在 PC 分辨率下隐藏文字标签**）
- 侧边栏不可在 PC 分辨率（1280px+）下被折叠成图标栏或隐藏

### 2.4 主内容区规范

- 主内容区使用 CSS Grid 布局，充分撑满剩余宽度
- KPI 卡片网格：使用 `repeat(auto-fit, minmax(200px, 1fr))` 自动填充，PC 下应呈现 **4~5 个横排 KPI**
- 内容面板之间间距：24px（`--space-lg`）
- 面板本身 `border-radius: 16px`，不使用超过 16px 的圆角

## 3. 响应式策略

### 3.1 策略定义：**PC 优先，无移动端支持**

本控制台是 **PC 专属工具**，不是面向移动端的响应式 Web App。响应式策略只处理 PC 宽窗口之间的弹性，不向下适配到手机。

### 3.2 断点规则（必须严格执行）

| 断点 | 规则 | 备注 |
|------|------|------|
| ≥ 1600px | 检查器展开，侧边栏 240px，内容区充分利用空间 | 理想状态 |
| 1400px ~ 1599px | 检查器宽度缩为 220px | 轻微压缩 |
| 1280px ~ 1399px | 检查器折叠隐藏，侧边栏保持 240px，内容区撑满 | 允许 |
| < 1280px | **不保证正确显示**，可以出现水平滚动条 | 正常行为 |

### 3.3 明确废除的响应式行为

以下当前代码中的响应式行为**必须废除或限制不触发**：

| 当前行为 | 问题 | 处理方式 |
|----------|------|----------|
| `@media (max-width: 720px)` 将侧边栏变为顶部横向导航 | 这是纯手机模式，不应在 PC 出现 | **删除此断点** |
| `@media (max-width: 720px)` 隐藏导航文字标签 | PC 下不应隐藏 | **删除** |
| 在 1200px 以下将双列布局强制变为单列 | 在 1280px PC 分辨率下过于保守 | 调整为 1100px 以下才单列 |

### 3.4 viewport meta tag

保持当前值不变：`<meta name="viewport" content="width=device-width, initial-scale=1.0" />`  
**不要**改成 `width=1280` 等固定值，PC 浏览器不需要这个。

## 4. 信息密度与层级要求

### 4.1 PC 大屏信息密度标准

PC 控制台应展示**足够的信息密度**，不能像移动端 App 一样大字大间距、每屏只显示几个元素。

| 区域 | 信息密度要求 |
|------|------------|
| KPI 区 | 一行展示 4~5 个 KPI 卡片，不换行 |
| Agent 卡片网格 | 至少每行 2~3 张，不可全部单列排布 |
| 任务列表 | 单行显示 Agent/标题/状态/时间，每屏可见 8~12 条 |
| 导航侧边栏 | 图标 + 文字同时可见，不折叠 |
| 面板行高 | 32px ~ 44px，不超过 52px（移动端常见大行高禁止） |

### 4.2 字号层级

| 层级 | 字号 | 用途 |
|------|------|------|
| 页面标题 | 20px ~ 24px | 当前分区名称 |
| 面板标题 | 14px, weight 700 | 各功能模块标题 |
| 正文内容 | 13px ~ 14px | 列表项、描述文字 |
| 辅助说明 | 12px | 时间戳、次要标签 |
| KPI 数值 | 28px ~ 32px | 大数字展示 |
| 极小标签 | 11px | UPPERCASE eyebrow 标签 |

**禁止**：正文字号超过 16px，行高超过 1.8，这会让页面稀疏如手机 App。

### 4.3 颜色层级要求（暗色主题）

- 背景色/面板色/卡片色需有**明显的深度差**（至少 3 层：页面底色 → 面板色 → 卡片悬停色）
- 主色（蓝色系）应饱和、明显，在深色背景上对比度 ≥ 4.5:1
- 禁止使用纯白文字（`#ffffff`）在深背景上，使用 `#e4ebff` 或类似暖蓝偏白
- 状态色（成功/警告/错误/信息）必须语义化，不能滥用同一种颜色

## 5. 验收标准清单

拉克丝和伊泽瑞尔完成整改后，产品经理艾克按以下清单逐条验收：

### 5.1 布局结构验收（必须全部通过）

- [ ] **L1**: 在 1440px 宽度下，侧边栏/内容区/检查器三栏均可见，无元素溢出或隐藏
- [ ] **L2**: 在 1280px 宽度下，侧边栏和内容区两栏正常展示，检查器折叠（可手动展开）
- [ ] **L3**: 在 1440px 下收起检查器后，内容区宽度正确撑满，无白边或空缺
- [ ] **L4**: 无任何断点触发"侧边栏变顶部横栏"或"导航文字消失"（这是手机模式）
- [ ] **L5**: 在 1280px 下水平方向无滚动条（内容自然适配），在 1200px 以下出现横向滚动是允许的

### 5.2 信息密度验收

- [ ] **D1**: Overview 页 KPI 区在 1440px 下呈现 4 个或以上横排卡片
- [ ] **D2**: Agents 页卡片网格在 1440px 下每行至少 2 个卡片（不是单列竖排）
- [ ] **D3**: 任务列表单行行高 ≤ 48px，每屏可见至少 8 条任务
- [ ] **D4**: 侧边栏导航文字在 1280px 以上始终可见（不隐藏）
- [ ] **D5**: 页面不出现大块留白（超过 200px 的无内容区域）

### 5.3 视觉质量验收

- [ ] **V1**: 打开控制台后整体视觉风格统一（无撕裂感、无浅色深色混用）
- [ ] **V2**: 主色（蓝色）在深背景下清晰可见，对比度满足基本可读性
- [ ] **V3**: KPI 数字与标签有明显字号层级差异（数字更大、更粗）
- [ ] **V4**: Active 导航项有明显高亮状态（区别于 hover 状态）
- [ ] **V5**: 面板/卡片有轻微阴影或边框，与背景有视觉分层

### 5.4 交互行为验收

- [ ] **I1**: 全站搜索（Cmd+K）弹出浮层位置居中，不贴屏幕底部
- [ ] **I2**: Toast 通知在右上角弹出，不遮挡主要操作区域
- [ ] **I3**: 检查器侧边栏展开/折叠动效流畅（不闪烁）
- [ ] **I4**: 所有可点击元素有 hover 反馈（背景色变化或边框变化）
- [ ] **I5**: 页面初始加载后，1440px 宽度下不出现横向滚动条

### 5.5 技术规范验收

- [ ] **T1**: `body` 或 `.shell` 设置了 `min-width: 1280px`（在浏览器 DevTools 可验证）
- [ ] **T2**: CSS 中不存在针对 `≤ 768px` 或 `≤ 720px` 的布局断点（移动端断点已删除）
- [ ] **T3**: `style.css` 中颜色全部使用 CSS 变量（`var(--xxx)`），无硬编码颜色散落

## 6. 不能做的事（Don'ts）

以下设计行为**明确禁止**，拉克丝和伊泽瑞尔的实现中均不得出现：

### 6.1 布局禁忌

| 禁止行为 | 原因 |
|----------|------|
| ❌ 侧边栏在任何 PC 分辨率下折叠为图标栏 | 这是移动端模式，PC 用户需要文字标签 |
| ❌ 侧边栏变为顶部横向导航栏 | 彻底手机化的布局，PC 控制台不可使用 |
| ❌ 内容区使用 `max-width: 768px` 或更小的宽度限制 | 会让宽屏出现大量留白，像手机界面居中 |
| ❌ 卡片/列表单列强制排布（在 1440px 下） | 浪费水平空间，信息密度不足 |
| ❌ 使用 `position: fixed; width: 100%; max-width: 480px` 之类的移动弹窗模式 | 纯手机 modal 尺寸 |

### 6.2 视觉禁忌

| 禁止行为 | 原因 |
|----------|------|
| ❌ 正文字号 > 16px | 会让文字密度极低，像放大字体的老人模式 |
| ❌ 行高 > 1.8（普通列表项） | 行间距过大，信息密度不足 |
| ❌ 卡片圆角 > 20px | 过于圆润，像移动端 App 卡片 |
| ❌ 内容区内边距 > 40px | 内容区缩窄，浪费屏幕空间 |
| ❌ 浅色和深色主题元素混用（同一个页面） | 样式割裂，视觉混乱 |
| ❌ 颜色系统中使用超过 3 套颜色变量体系 | 当前旧代码已有此问题，必须统一 |

### 6.3 响应式禁忌

| 禁止行为 | 原因 |
|----------|------|
| ❌ 添加 `@media (max-width: 720px)` 断点 | 手机断点，本工具不支持手机 |
| ❌ 添加 `@media (max-width: 480px)` 断点 | 同上 |
| ❌ 使用 `user-scalable=no` | 可访问性违规 |
| ❌ 不设 `min-width` 并假设内容"会自适应" | 没有 min-width 限制会在窄窗口呈现手机效果 |

## 7. 整改优先级说明

拉克丝和伊泽瑞尔应按以下优先级完成整改：

| 优先级 | 任务 | 说明 |
|--------|------|------|
| P0 🔴 | 加 `min-width: 1280px` | 立刻解决"手机比例"问题的根本原因 |
| P0 🔴 | 删除 `@media (max-width: 720px)` 移动端断点 | 消除手机侧边栏折叠行为 |
| P0 🔴 | 修复 1440px 下 KPI 横排（4个+） | 信息密度基线 |
| P1 🟡 | 统一颜色变量体系（清除旧套颜色） | 消除视觉割裂 |
| P1 🟡 | 检查器默认展开（1440px+） | 充分利用空间 |
| P2 🟢 | 字号层级细化 | 提升专业感 |
| P2 🟢 | 交互动效统一 | 提升流畅度 |

## 附录：当前问题根本原因分析

经过代码审查，李琪反馈的三个问题根本原因如下：

1. **配色糟糕**：拉克丝上一版已重建颜色系统（见 `2026-03-19-ui-redesign-lux.md`），理论上已修复。如果仍觉得糟糕，可能是浏览器缓存未更新，或存在新引入的内联样式覆盖了 `style.css`。

2. **PC 控制台呈手机比例**：根本原因是 `body` 和 `.shell` 均无 `min-width` 限制，同时 `@media (max-width: 720px)` 断点会在窗口缩小时将侧边栏变为横向顶栏、隐藏文字标签。若用户浏览器窗口或系统缩放未到 100%，可能触发此断点。

3. **不支持自适应**：当前代码的响应式断点存在，但响应式逻辑走向是"向下压缩到手机"，而不是"PC 宽度内弹性适配"。核心问题是响应式策略目标不对，不是没有响应式。
