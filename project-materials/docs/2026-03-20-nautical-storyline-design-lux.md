## 一、故事背景设定

### 世界观（核心叙事）

> **在无尽的数字海域中，一支精锐船队扬帆远航。每一次任务都是一场航程——船长统筹航线，领航员锁定目标，船匠铸造龙骨，水手们各司其职，直到安全抵达彼岸。没有孤胆英雄，只有协作远航。**

（78字）

### "船队"与"AI Agent 协作团队"的类比

| 航海概念 | 系统映射 |
|----------|----------|
| 船队 | 多 Agent 协作团队 |
| 一次航程 | 一个项目/任务生命周期 |
| 目的港 | 项目交付目标 |
| 航线规划 | 需求分析与优先级排序 |
| 船体结构 | 系统架构设计 |
| 船舱分工 | 模块划分与角色职责 |
| 海图 | PRD / 设计文档 |
| 航行日志 | 过程文档、决策记录 |
| 靠港验收 | 联调测试与交付验收 |
| 灯塔 | 技术顾问的跨域指引 |

### "任务执行"到航海叙事的映射

1. **接到任务** → 收到远航委托，确认目的港
2. **需求拆解** → 领航员绘制海图，标注航线关键节点
3. **架构设计** → 船匠确定龙骨与舱位结构
4. **开发实现** → 水手们各就各位，扬帆起航
5. **联调测试** → 通过风暴区前的全船检修
6. **交付上线** → 安全抵达目的港，卸货交接
7. **复盘** → 更新航行日志，为下次远航积累经验

### 命名设计原则

1. **名称即职责**：每个名字直接关联其航海功能（赫尔姆=舵轮，诺斯=北方/方向，基尔=龙骨，帆索=帆与索具，锚铁=锚，望潮=观察潮汐，星绘=星图绘制，顺风=有利风向，灯塔=指引）
2. **中文优先**：两字名便于日常称呼，也可用职称
3. **英文标识简洁**：适用于 agent ID、配置文件、代码引用
4. **无版权风险**：均为通用航海概念词汇组合，不涉及任何 IP

### 角色详细设定

#### 🧭 赫尔姆 · 船长（原提莫）
统领全船的核心指挥者。掌舵方向，调度人力，确保每一段航程安全抵达。所有船员向船长汇报，所有与委托方的沟通由船长统一对接。

#### 🗺️ 诺斯 · 领航员（原艾克）
确定航线的战略角色。接收远航委托后，绘制航线图、标注关键航段、定义优先顺序。是"去哪里"和"先去哪里"的决策者。

#### 🔩 基尔 · 船匠（原贾克斯）
船体结构的设计与守护者。定义龙骨走向、舱位划分和承重边界。确保整条船在航行中不散架，在扩建时有空间。

#### ⛵ 帆索 · 帆手（原伊泽瑞尔）
驾驭风力、控制船帆的实施者。负责船只上层建筑与甲板上所有用户可见的操控面。让航行顺滑、响应灵敏。

#### ⚓ 锚铁 · 轮机长（原雷欧娜）
船体动力与底层机械的守护者。负责引擎、管路、货舱——一切在甲板之下驱动船只运转的系统。

#### 🔭 望潮 · 瞭望员（原加里奥）
站在桅杆顶端的眼睛。负责在靠港前发现暗礁和漏洞，确保航路安全、货物完好。全船最后一道质量防线。

#### ✨ 星绘 · 海图师（原拉克丝）
绘制海图、标注航线、设计船舱内部布局的视觉专家。让每一份图纸清晰易读，让船上每个角落井然有序。

#### 📦 顺风 · 补给官（原杰斯）
管理船队物资与后勤效率的角色。优化补给流程、维护工具库、确保船员有称手的装备。让航行少摩擦，多效率。

#### 🏮 灯塔 · 灯塔守望者（原瑞兹）
不在船上，在岸上。远望全域海况，为迷航的船只提供方向指引。跨越多条航线的经验沉淀者和技术顾问。

## 三、层级结构映射

| 原层级 | 系统职责 | 航海映射 | 隐喻 |
|--------|----------|----------|------|
| L0 全局巡检层 | 跨项目健康监控 | **🏮 灯塔层（Lighthouse）** | 灯塔从海岸高处俯瞰全部航线，发出预警和指引 |
| L1 项目总控层 | 单项目指挥调度 | **🧭 舰桥层（Bridge）** | 船长在舰桥统筹全船，发号施令 |
| L2 专业角色层 | 具体任务执行 | **⚓ 甲板层（Deck）** | 各岗位船员在甲板各司其职，协作航行 |

### 层级叙事

- **灯塔层**：岸上最高处，照亮所有航线。不参与具体航行，但随时为任何一条船提供方向校准和风险预警。
- **舰桥层**：每条船的指挥中枢。船长和领航员在此制定航线、调度资源、做出关键决策。
- **甲板层**：船上的执行面。帆手操帆、轮机长管引擎、瞭望员盯海面、海图师绘图纸、补给官调物资——各有分工，协同推进。

## 五、与英雄联盟体系的对比优势

### 1. 版权风险

| 维度 | 英雄联盟体系 | 航海探险体系 |
|------|-------------|-------------|
| IP 归属 | 提莫、艾克等为 Riot Games 注册商标 | 航海概念为公共领域，无版权归属 |
| 商用风险 | 对外传播或商业化时存在侵权风险 | 完全安全，可自由商用和传播 |
| 形象限制 | 角色形象受原作设定约束 | 可自由定义视觉形象和个性 |

### 2. 叙事一致性（协作 vs 对抗）

| 维度 | 英雄联盟体系 | 航海探险体系 |
|------|-------------|-------------|
| 原始语境 | 5v5 对抗竞技 | 协作远航、共同抵达 |
| 角色关系 | 原作中可能是对手 | 天然是同一条船上的队友 |
| 任务隐喻 | "打赢对面"（竞争） | "到达目的港"（交付） |
| 失败隐喻 | "被击败" | "遇到风暴需要调整航线"——更温和、更符合工作实际 |

### 3. 办公场景契合度

| 维度 | 英雄联盟体系 | 航海探险体系 |
|------|-------------|-------------|
| 职级映射 | 游戏中无明确层级 | 船长→船员的层级清晰自然 |
| 任务流程 | 游戏是实时对抗，无阶段 | 航海有出发→航行→抵达的完整流程 |
| 专业分工 | 游戏角色可互换位置 | 航海岗位天然专业化且不可替代 |
| 正式感 | 游戏感强，办公场景略显违和 | 航海探险自带专业感和使命感 |

### 4. 对外传播友好度

| 维度 | 英雄联盟体系 | 航海探险体系 |
|------|-------------|-------------|
| 认知门槛 | 需要了解英雄联盟才能get到梗 | 航海是全球通用隐喻，零门槛理解 |
| 受众范围 | 偏向游戏玩家群体 | 适合所有职场人群 |
| 国际化 | 英雄联盟在不同地区知名度不一 | 航海探险是跨文化的通用叙事 |
| 专业印象 | 可能被认为"不够正式" | 航海探险传达冒险精神和专业协作 |

## 七、视觉风格建议（后续迭代）

> 以下为后续视觉设计的方向性建议，具体规范待独立设计任务展开。

- **主色调**：深海蓝（#1B3A5C）+ 罗盘金（#C8A45A）+ 白帆色（#F5F0E8）
- **辅助色**：风暴灰（#6B7B8D）、灯塔橙（#E8824A）、海图棕（#8B6F47）
- **图标风格**：线性描边，航海器物（罗盘、锚、帆、望远镜、灯塔）
- **字体建议**：标题用有力度的无衬线体，正文用清晰的等宽/无衬线体
- **整体氛围**：专业、沉稳、有探险精神——不是海盗的粗犷，而是远洋船队的精密与信任
