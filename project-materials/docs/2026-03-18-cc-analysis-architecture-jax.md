## 可借鉴点汇总

### 1. 项目目录分层

| 当前 ODA | 建议借鉴 CC 方式 |
|----------|-----------------|
| 模块边界模糊 | 清晰分层：`src/adapters`（外部适配）/ `src/clients`（网关客户端）/ `src/runtime`（核心逻辑）/ `src/ui`（界面服务） |

### 2. 实时交互策略

| 当前 ODA | 建议借鉴 CC 方式 |
|----------|-----------------|
| SSE 推送，无多级降级 | 配置化轮询间隔（会话列表 10s / 状态更新 2s）+ 故障降级为快照模式 |

### 3. 数据流与状态管理

| 当前 ODA | 建议借鉴 CC 方式 |
|----------|-----------------|
| 数据分散，无统一快照 | 增量快照存储（`last-snapshot.json`）+ 差异计算 + 本地存储库（projects/tasks/budgets/ui-preferences） |

### 4. 页面加载性能

| 当前 ODA | 建议借鉴 CC 方式 |
|----------|-----------------|
| 未优化 | 服务端渲染 HTML 直接返回 + `compact` 参数减少数据 + 基于时间窗口的分页加载 |

### 5. 错误处理与监控

| 当前 ODA | 建议借鉴 CC 方式 |
|----------|-----------------|
| 错误处理分散 | 统一 `x-request-id` 追踪 + 多层审计日志（timeline / operation-audit / approval-actions）+ `/healthz` 端点检查快照新鲜度 |

### 6. 关键痛点解决方案

| 痛点 | CC 方案 |
|------|---------|
| 写操作安全性 | 默认只读 + 显式环境变量开启 + dry-run 开关（**我们已实现**） |
| 搜索 | 多实体搜索 API（/search/tasks / /projects / /sessions），安全子串匹配，限制结果数 |
| Agent 状态追踪 | sessionsList + sessionStatus + 本地缓存，6种状态分类 |
| 设计 Token | `ui-preferences.json` 持久化用户偏好 + 快速过滤器 |
| 权限控制 | 本地 token 认证头 `x-local-token`（**我们已实现类似方案**） |

### 核心架构原则（来自 CC）

> "官方接口优先 + 薄适配层"：通过本地代理网关模式安全读取，所有高风险操作默认禁用需显式配置，数据流转以快照存储为中心，支持回溯和状态对比。

---

## 建议我们立即借鉴的 3 项

1. **`/healthz` 健康检查端点** — 检查快照新鲜度、Gateway 延迟、关键文件可读性
2. **`x-request-id` 全链路追踪** — 便于排查请求异常
3. **`compact` 参数支持** — 在请求量大时允许前端请求精简版数据，减少带宽
