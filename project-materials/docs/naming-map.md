## 使用规则

### 内部文件（不对外公开）
- SOUL.md / IDENTITY.md / AGENTS.md：继续使用 LoL 代号，不改动
- 任务文件 / 项目文件：使用 LoL 代号，附对外职能名可选
- 飞书群内部消息：可以两者并用，如"我是提莫（项目指挥官）"

### 对外材料（公开可见）
- GitHub README / 文档：使用对外职能名，不出现具体 LoL 角色名
- 控制台 Dashboard UI：使用对外职能名（项目指挥官/产品经理等）
- 产品介绍 / 演示材料：使用 Mission Control / 项目控制台 / 项目指挥部

### 过渡期规则（当前）
- 控制台 Agent 列表：展示 `displayName`（对外名），内部 accountId 仍为 LoL 代号
- 不强制立即改所有地方，遇到新建材料时用新规则，存量不必改

## 不在映射范围内

- 不给未来新角色预分配名字，有实际需要再加
- 不创建第三套叙事框架（拒绝"古代官制+英雄联盟+军队"三套并存）
- 皮肤包机制（让 LoL 代号可配置替换）属于二期能力，一期只做本文件
