# OpenClaw Office Console（Windows 版）首页与核心页面字段清单
> 基于 MVP 架构稿继续细化的页面字段级说明

---

## 1. 文档目的

本文档用于把 **OpenClaw Office Console（Windows 版）** 的首页及核心页面进一步细化到字段级，方便后续开展：

- 原型设计
- 前端页面拆解
- 后端接口设计
- 状态管理设计
- 开发任务拆分

本文档聚焦第一版 MVP，不扩展到过重的高级能力。

---

## 2. 页面范围

本次细化的核心页面包括：

1. 欢迎页 / 首启入口页
2. 环境检测页
3. 模型配置页
4. 飞书配置页
5. 首页总览
6. Agent 中心
7. 任务中心
8. 任务详情页
9. 日志与诊断页

---

## 3. 页面设计总原则

### 3.1 优先展示“当前可行动信息”
页面应优先展示：

- 当前状态
- 当前缺失项
- 当前推荐动作
- 当前错误与处理方式

### 3.2 避免技术术语压迫感
前台避免优先使用：

- planner
- executor
- workflow runtime
- tool graph
- orchestration

优先使用业务化语言：

- 助手
- 当前任务
- 当前步骤
- 检测结果
- 建议操作
- 重试

### 3.3 页面必须支持“下一步”
每个关键页面都应至少存在一类明确动作：

- 去配置
- 测试连接
- 重新检测
- 重试任务
- 查看详情
- 安装能力

# 4. 欢迎页 / 首启入口页

## 4.1 页面目标
用于帮助用户确认当前是：

- 新安装
- 关联已有 OpenClaw

并建立对控制台的基本认知。

## 4.2 页面字段

### A. 顶部标题区
- 产品名称：OpenClaw Office Console
- 副标题：面向办公用户的 OpenClaw 可视化控制台
- 当前版本号

### B. 简介区
- 简介文案
- 主要能力摘要（模型配置、飞书接入、Agent 管理、任务状态、错误反馈）

### C. 入口选择区
#### 卡片 1：新安装
字段：
- 标题：新安装
- 描述：首次在本机初始化 OpenClaw 办公控制台
- 按钮：开始新安装

#### 卡片 2：关联已有实例
字段：
- 标题：关联已有实例
- 描述：关联本机已存在的 OpenClaw 环境与工作目录
- 按钮：关联已有实例

### D. 底部辅助区
- 查看安装说明
- 查看系统要求
- 打开高级后台（可选，弱化）

# 5. 环境检测页

让用户看懂当前设备是否满足运行条件，并知道哪些问题需要先处理。

## 5.2 页面布局
- 顶部步骤导航
- 中部检测结果卡片列表
- 底部操作区

## 5.3 检测项字段定义

每一个检测项都建议统一字段结构：

- 检测项名称
- 状态（通过 / 警告 / 失败）
- 检测结果摘要
- 详细说明
- 修复建议
- 操作按钮

## 5.4 第一版推荐检测项

### 1. 操作系统
字段：
- 系统名称
- 系统版本
- 是否满足最低要求
- 状态

### 2. OpenClaw 环境
字段：
- 是否检测到已有 OpenClaw
- 检测路径
- 可否关联
- 状态

### 3. 工作目录可写性
字段：
- 默认工作目录路径
- 是否可写
- 状态
- 修改目录按钮

### 4. 端口占用
字段：
- 默认控制台端口
- 默认后端端口
- 是否冲突
- 状态
- 重新检测按钮

### 5. 浏览器能力
字段：
- 浏览器连接状态
- 浏览器检测结果
- 最近检测时间
- 状态
- 重新检测按钮

### 6. 本地网络状态
字段：
- 基础联网状态
- 模型接口可达性（初步）
- 状态

## 5.5 页面底部动作
- 上一步
- 重新检测
- 继续下一步

# 8. 首页总览

## 8.1 页面目标
让用户第一次进入主界面时，立即回答四个问题：

1. 我现在系统状态如何
2. 我缺什么配置
3. 我可以用哪些 Agent
4. 当前任务做到哪一步

## 8.2 页面结构
- 顶部状态摘要栏
- Console Agent 建议区
- Agent 快速入口区
- 当前任务与最近任务区
- 最近结果与更新动态区

---

## 8.3 顶部状态摘要栏字段

### 状态项 1：模型状态
- 是否已配置
- 是否最近测试通过
- 当前默认模型
- 点击动作：进入模型配置页

### 状态项 2：飞书状态
- 是否已连接
- 是否测试写入成功
- 点击动作：进入飞书配置页

### 状态项 3：浏览器状态
- 是否正常
- 最近检测时间
- 点击动作：进入检测或修复页

### 状态项 4：Workspace 状态
- 当前工作目录
- 是否可写
- 点击动作：进入基础设置页

### 状态项 5：Agent 健康
- 正常 Agent 数
- 异常 Agent 数
- 点击动作：进入 Agent 中心

### 状态项 6：任务摘要
- 今日成功任务数
- 今日失败任务数
- 点击动作：进入任务中心

---

## 8.4 Console Agent 建议区字段

### A. 当前摘要
- 一段系统当前概述

### B. 发现的问题
- 1~3 条全局问题摘要

### C. 推荐下一步
- 1 条主推荐动作
- 可附带 1 条次级推荐

### D. 最近更新
- 1~3 条更新提示

### E. 操作按钮
- 去完成配置
- 体验推荐 Agent
- 查看全部建议

---

## 8.5 Agent 快速入口区字段

- 最近结果：最近一次未执行
- 主按钮：去配置 / 立即试用
- 次按钮：查看说明

#### 文档初稿助手
- 名称
- 描述：快速生成结构化文档初稿
- 状态
- 前置条件
- 最近结果

# 13. 页面之间的关键跳转关系

## 13.1 首页 → 配置中心
通过状态摘要卡片或 Console Agent 建议区跳转。

## 13.2 首页 → Agent 中心
通过 Agent 快速入口区点击进入。

## 13.3 首页 → 任务中心
通过任务摘要或当前任务点击进入。

## 13.4 任务中心 → 任务详情页
点击任务行进入详情。

## 13.5 日志与诊断 → 配置中心
通过修复建议中的相关按钮跳转。

# 15. 总结

这套页面字段清单服务于一个明确原则：

> 第一版不是做“功能最多”的平台，而是做“看得懂、配得上、跑得通、错了能处理”的办公控制台。

因此页面设计的核心不在于堆功能，而在于把以下信息稳定地呈现出来：

- 当前状态
- 缺失项
- 推荐动作
- 正在做什么
- 为什么失败
- 怎么继续
