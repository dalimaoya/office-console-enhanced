# OpenClaw Office Console（Windows 版）MVP 页面原型文案稿
> 基于 MVP 架构稿继续细化的页面文案与交互草稿

---

## 1. 文档目的

本文档用于给 **OpenClaw Office Console（Windows 版）** 第一版核心页面提供：

- 页面标题
- 页面说明文案
- 状态提示文案
- 按钮文案
- 空状态文案
- 错误提示文案
- 推荐动作文案

本稿主要用于：

- 画低保真 / 中保真原型
- 做桌面应用 Demo
- 统一产品语气
- 前端实现时直接引用

---

## 2. 文案原则

### 2.1 语气原则
- 客观
- 克制
- 易懂
- 直接指向动作
- 避免夸张营销感

### 2.2 表达原则
优先说清楚：
- 现在是什么状态
- 影响了什么
- 下一步做什么

避免：
- 大量技术术语
- 空泛鼓励
- 无动作建议的提示

# 15. 总结

第一版页面文案的核心作用不是“让产品显得聪明”，而是：

- 降低理解成本
- 减少用户焦虑
- 让用户知道当前状态
- 让用户知道下一步该做什么

因此文案必须始终围绕三件事组织：

1. 现在是什么状态
2. 影响了什么
3. 下一步做什么
